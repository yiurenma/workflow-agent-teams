## 1. 产品描述

**Workflow 平台** 支持对交易类消息进行 **可配置丰富化、条件判断与多通道下发**，在少写定制后端的前提下完成集成与运维。

三大功能线：

| 功能线 | 前缀 | 范围 |
|--------|------|------|
| 应用管理 | **APP** | 在 `/workflows` 的应用注册、设置、复制、配置历史与生命周期 |
| 运行记录管理 | **REC** | 执行记录的浏览、筛选、下钻；运行结果在记录中的含义（用户与对接方可理解） |
| 画布 | **CV** | `/workflows/$applicationName` 可视化编排、规则（JSONPath）、保存、试跑、Explain |

配置态与运行态职责分离、数据贯通。对接方如何调用、返回码表、网络协议等 **不在本文展开**。

## 2. 用户与角色

| 角色 | 侧重 |
|------|------|
| 集成工程师 | **APP** + **CV** |
| 应用开发者 | **REC**（提交请求与幂等、重试等 **业务语义**，非接口手册） |
| 运维 / QA / 分析 | **REC** + **CV**（试跑） |
| 产品 / 业务（偏只读） | **CV**（Explain） |

### 【REC】3.2 运行记录管理

#### REC-US-19 — 浏览、筛选、下钻执行记录

**桌面：** REC-AC-19-D1 独立页/入口、列字段满足业务核对 · REC-AC-19-D2 筛选、分页、纵滚、总数 · REC-AC-19-D3 确认号、跟踪号、客户 ID 等在默认策略下可见  

**移动：** REC-AC-19-M1 同等筛选维度 · REC-AC-19-M2 列表/卡片可浏览、关键字段优先 · REC-AC-19-M3 详情与桌面承诺一致  

**通用：** REC-AC-19-G1 父子/重试链可理解 · REC-AC-19-G2 管理端能展示记录数据（数据来源与查询方式由架构/实现定义）；不削弱 **CV**  

#### REC-US-12 — 业务系统提交运行请求（产品语义）

*与终端形态无关。接口形态、报文格式、错误码 **见架构师文档**。*

**验收标准：** REC-AC-12-1 对接方可以按约定方式提交运行请求（负载格式、关联标识、业务确认信息等以 **发布给对接方的说明** 为准，不在本产品正文列 URL）。  

REC-AC-12-2 同步/异步行为与应用侧配置一致，且对调用方 **有书面说明、可验证**。  

REC-AC-12-3 重复提交同一业务键时系统 **不重复执行业务处理**；未知应用、重复键等场景下返回 **可分类、可沟通** 的失败说明（具体码表见架构文档）。  

#### REC-US-13 — 丰富化阶段（记录可解释）

*与终端无关。*

REC-AC-13-1 丰富化步骤在派发前按 **画布连线决定的顺序** 执行。  

REC-AC-13-2 每步规则为 JSONPath **全部匹配** 才执行；结果合并进运行时负载（字段含义见架构文档）。  

REC-AC-13-3 条件类步骤仅在规则匹配时应用其逻辑配置；不匹配则跳过，**无**隐式默认分支。  

REC-AC-13-4 丰富化结果在记录总状态上 **可被运维理解**。  

#### REC-US-14 — 下发阶段（记录可解释）

*与终端无关。*

REC-AC-14-1 下发在丰富化完成后执行。  

REC-AC-14-2 跟踪信息与各通道状态写入记录；最终总状态反映组合结果。  

REC-AC-14-3 通道模型可扩展未来类型。  

#### REC-US-15 — 幂等（产品语义）

*与终端无关。*

REC-AC-15-1 同一应用下同一业务关联键在系统中唯一。  

REC-AC-15-2 重复请求被识别并拒绝重复处理，且不产生重复业务后果。  

REC-AC-15-3 约定条件下的重试提交可按规则绕过重复校验（细节见架构文档）。  

#### REC-US-16 — 失败重试（产品语义）

*与终端无关。*

REC-AC-16-1 命中应用级重试策略时产生可关联的重试记录。  

REC-AC-16-2 重试次数可追溯；用尽后终态明确。

#### CV-US-32 — 节点编辑器无障碍访问（屏幕阅读器）

> **作为** 使用屏幕阅读器编辑工作流节点的用户，**我希望** 节点编辑器抽屉在打开时能够宣告其用途，**以便** 我理解正在交互的区域并能高效导航表单。

**验收标准：** CV-AC-32-1 节点编辑器抽屉容器元素具有 `aria-label="Node Configuration"` · CV-AC-32-2 抽屉打开时，屏幕阅读器（NVDA、JAWS、VoiceOver）宣告"节点配置"或等效本地化文本 · CV-AC-32-3 修复适用于桌面（420px 宽抽屉）和移动（全宽抽屉）视口 · CV-AC-32-4 Axe DevTools 或类似无障碍扫描器报告抽屉上缺少 `aria-label` 的零违规

#### CV-US-33 — 节点编辑器标题文本符合 WCAG AA 对比度要求

> **作为** 低视力或色盲用户，**我希望** 节点编辑器标题文本与背景有足够的对比度，**以便** 我可以轻松阅读节点类型和配置标签而不费力。

**验收标准：** CV-AC-33-1 标题文本颜色从当前值更改为 `#6b6b75` 或更深，以实现至少 4.5:1 的对比度 · CV-AC-33-2 使用 WebAIM 对比度检查器或浏览器开发工具验证对比度，符合 WCAG 2.1 AA 级标准 · CV-AC-33-3 新颜色适用于节点编辑器标题中的所有文本（节点类型标签、章节标题）· CV-AC-33-4 桌面和移动视口均满足对比度要求 · CV-AC-33-5 Axe DevTools 或类似无障碍扫描器报告节点编辑器抽屉中的零颜色对比度违规

#### CV-US-34 — 节点编辑器表单字段描述文本符合 WCAG AA 对比度要求

> **作为** 视力受损或在强光环境下工作的用户，**我希望** 节点编辑器中的表单字段描述文本具有足够的颜色对比度，**以便** 我可以轻松阅读指导文本而不费力。

**验收标准：** CV-AC-34-1 表单字段描述文本颜色从 `text-zinc-400` (#a1a1aa) 更改为 `text-zinc-600` (#52525b)，以实现至少 4.5:1 的对比度 · CV-AC-34-2 修复应用于 HttpCallForm 和 LogicForm 组件中的以下 3 个描述段落："The name shown on the canvas..."（节点名称描述）、"Run only when..."（规则描述）、"What the system does..."（动作描述）· CV-AC-34-3 在白色 (#ffffff) 和浅灰色 (#fafafa) 背景上对比度均达到或超过 4.5:1 · CV-AC-34-4 视觉外观与整体设计系统保持一致（文本仍显示为辅助/帮助文本，而非主要内容）· CV-AC-34-5 受影响组件无其他样式更改

#### CV-US-37 — 节点编辑器抽屉关闭按钮

> **作为** 编辑工作流节点的用户，**我希望** 节点编辑器抽屉的 × 关闭按钮能可靠地关闭抽屉，**以便** 我可以无障碍地返回画布。

#### CV-US-38 — Playwright 五层 UX 验证框架

> **作为** 运行 E2E 测试的 QA 工程师，**我希望** Playwright 测试在五个层面（存在、尺寸、视口、交互、效果）验证 UI 正确性，**以便** 在 UAT 之前捕获视觉回归和 UX 缺陷。

**验收标准：** CV-AC-38-1 所有 Playwright 测试用例使用五层验证框架（Layer 1 存在、Layer 2 尺寸、Layer 3 视口、Layer 4 交互、Layer 5 效果）· CV-AC-38-2 改造现有测试用例以包含 Layer 2-5 检查 · CV-AC-38-3 为关键 UI 状态捕获截图基线并在 CI 中比对 · CV-AC-38-4 测试报告记录每个测试用例验证了哪些层

#### CV-US-39 — 审计并修复残留的非 Carbon UI

> **作为** Workflow Studio 用户，**我希望** 所有 UI 组件一致遵循 IBM Carbon Design Language，**以便** 界面感觉连贯且专业。

**验收标准：** CV-AC-39-1 审计识别所有不符合 IBM Carbon 规范的 UI 元素 · CV-AC-39-2 审计产生带根本原因的编号差距列表 · CV-AC-39-3 修复所有识别的差距 · CV-AC-39-4 文档解释为什么 Playwright 最初遗漏这些样式差距并添加断言/截图基线 · CV-AC-39-5 应用删除确认对话框特别验证使用 Carbon 样式

#### CV-US-40 — 规则键必须是单个 JSONPath 并有清晰错误

> **作为** 配置节点规则的工作流作者，**我希望** 系统验证每个规则键是单个有效的 JSONPath 表达式，**以便** 如果语法错误我能立即得到反馈。

**验收标准：** CV-AC-40-1 验证规则键是单个 JSONPath（无逗号/分号，通过 jsonpath-plus 验证有效语法）· CV-AC-40-2 验证失败时显示清晰错误消息 · CV-AC-40-3 存在验证错误时保存按钮禁用 · CV-AC-40-4 验证在失焦和保存尝试时运行

#### CV-US-41 — 可调整大小的抽屉、只读默认、顶部编辑按钮

> **作为** 编辑节点的工作流作者，**我希望** 可调整大小的抽屉以只读模式打开并在顶部有编辑按钮，**以便** 我可以舒适地查看长配置并清楚区分查看和编辑模式。

#### CV-US-42 — 带单行描述的节点调色板

> **作为** 工作流作者，**我希望** 节点调色板为每种节点类型显示单行描述，**以便** 在将其添加到画布之前了解每个节点的作用。

#### CV-US-36 — 界面视觉设计系统（IBM Carbon Design Language）

> **作为** Workflow Studio 用户，**我希望** 界面采用 IBM Carbon Design Language 规范，**以便** 平台传达企业级权威感、系统性精确度与专业可信度。

**验收标准：** CV-AC-36-1 字体系统：IBM Plex Sans（主字体）+ IBM Plex Mono（代码/技术内容）；展示级标题（42px+）weight 300，正文 16px weight 400，紧凑正文 14px letter-spacing 0.16px，说明文字 12px letter-spacing 0.32px；强调使用 weight 600，不使用 weight 700；所有文本符合 WCAG 2.1 AA 对比度要求。

CV-AC-36-2 色彩系统：背景白色 `#ffffff`；主文本 Gray 100 `#161616`；次要文本 Gray 70 `#525252`；卡片/表面 Gray 10 `#f4f4f4`；边框 Gray 30 `#c6c6c6`；唯一强调色 IBM Blue 60 `#0f62fe`；链接悬停 Blue 70 `#0043ce`；焦点环 Blue 60 2px；错误 Red 60 `#da1e28`；成功 Green 50 `#24a148`；导航栏背景 Gray 100 `#161616`；不引入额外强调色，不使用渐变。

CV-AC-36-3 组件形状：按钮 0px 圆角（矩形）；输入框 0px 圆角仅底边框；卡片/磁贴 0px 圆角无阴影（通过背景色分层表达层级）；标签/状态徽章 24px 圆角（唯一圆角例外）。

CV-AC-36-4 按钮样式：主按钮 `#0f62fe` 背景 48px 高；次按钮 `#393939` 背景；幽灵按钮透明背景 `#0f62fe` 文字 1px 边框；危险按钮 `#da1e28` 背景。

CV-AC-36-5 表单输入：`#f4f4f4` 背景，40px 高，16px 水平内边距，仅底边框（默认透明 → 焦点 `#0f62fe` → 错误 `#da1e28`）。

CV-AC-36-6 导航栏：`#161616` 背景 48px 高；链接 14px weight 400 `#c6c6c6`；悬停 `#ffffff`；移动端汉堡菜单。

CV-AC-36-7 卡片与表格：卡片 `#f4f4f4` 背景悬停 `#e8e8e8`，0px 圆角无阴影；表格行高 48px，行底边框 `1px solid #e0e0e0`，悬停 `#f4f4f4`。

CV-AC-36-8 间距：基础单位 8px，所有间距为 8 的倍数（微调允许 2px/4px）；组件内边距 16px；章节间距 48px。

CV-AC-36-9 层级与阴影：卡片无阴影，通过背景色分层（白色 → `#f4f4f4` → `#e0e0e0`）表达深度；下拉/提示/浮层使用 `0 2px 6px rgba(0,0,0,0.3)`；模态框/抽屉加暗色遮罩。

CV-AC-36-10 功能回归：所有现有功能（应用管理、画布编辑、节点编辑器、记录、Explain、生成器）保持完整；无 API 变更；移动端响应式保留；WCAG 2.1 AA 无障碍要求维持。

#### CV-US-35 — 界面视觉设计系统（Quiet Luxury）

> **作为** Workflow Studio 用户，**我希望** 界面采用精致、温暖的视觉设计，**以便** 平台传达品质感与专业性，而非冷淡的技术感。

**验收标准：** CV-AC-35-1 字体系统：标题采用衬线或精致无衬线字体（如 Crimson Pro、Lora、Instrument Serif），正文采用高质量 UI 字体；建立清晰的字体层级（页面标题、章节标题、正文、辅助文本）；所有文本符合 WCAG AA 对比度要求（正文 4.5:1，大文本 3:1）。

CV-AC-35-2 色彩系统：用温暖中性色替换冷灰色（zinc-400/500/600）；主文本为温暖深灰（#2d2d2d 范围），次要文本为温暖中灰（#6b6b6b 范围），边框为柔和温灰（#d4d4d4 范围）；引入大地色调强调色（主操作：陶土/粘土/柔金，成功：鼠尾草绿/橄榄绿，警告：琥珀/赭石，错误：柔和锈红/砖红，信息：板岩蓝/温灰蓝）；背景为温白/奶油色（#fafaf8 范围）与浅沙/米色（#f4f3f0 范围）；所有颜色组合符合 WCAG AA 对比度要求。

CV-AC-35-3 间距与层级：关键区域增加留白（导航头部 16-20px 垂直内边距，应用列表卡片 20-24px 内边距，表格行高 48-56px，表单字段间距 16-20px）；建立清晰视觉层级（页面级 32-40px，组件级 16-24px，元素级 8-12px）；响应式行为保持，间距在移动端适当缩放。

CV-AC-35-4 组件精致化：导航头部（精致字体、温暖背景、柔和阴影）；应用列表桌面表格（精致边框、适当行高、温暖悬停态）；应用卡片移动端（柔和阴影、圆角 8-12px、温暖背景、清晰层级）；按钮（主按钮温暖强调色、次按钮轮廓/幽灵样式、精致过渡）；状态标签（柔和颜色、圆角、适当内边距）；画布节点（精致样式、柔和阴影、温暖背景、清晰字体、微妙悬停/选中态）；抽屉与模态框（温暖背景、精致标题、适当内边距与间距、柔和阴影）。

CV-AC-35-5 动效与过渡：一致的过渡时长（快速交互 150-200ms，中速交互 250-300ms，慢速交互 300-400ms）；精致缓动函数（优先 ease-out 或自定义 cubic-bezier）；一致应用于按钮悬停、抽屉/模态框开关、下拉菜单、画布交互。

CV-AC-35-6 视觉验证：从 UAT 环境（https://workflow-ui-gamma.vercel.app/workflows）捕获基线截图（应用列表桌面/移动、画布视图、节点编辑器抽屉、模态框、记录列表）；从本地开发捕获对比截图（相同视图）；在 `STYLE_REFACTOR_LOG.md` 中记录视觉变更（字体、色彩、间距、组件精致化、前后截图引用）。
