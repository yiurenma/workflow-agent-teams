## T2-001 — 画布左侧「节点」选项：可拖拽放置 + 每种节点一句话说明

**类型：** 画布 / 节点面板 / 文案与交互  
**标签（供检索）：** `T2-canvas-node-palette-descriptions-drag-drop`

### 背景

用户在画布（编排 / 发布视图）左侧看到可拖到画布上的节点列表时，**目前缺少说明**，不知道每种节点是干什么的，体验突兀。

### 期望行为

1. **左侧节点区**支持（或保持）**拖拽到画布**上添加节点；与移动端「添加节点」入口在能力上应对齐说明文案（同一套节点类型的描述）。
2. **每一种可拖拽的节点类型**都配有 **简短、面向用户的说明**（主文案一行即可；可选：悬停 tooltip 或「?」再展开一句）。
3. **示例文案方向**（需 PM 与研发核对运行时语义后定稿）：
   - **HTTP fetch**：通过 HTTP 请求获取数据，把响应结果带入工作流供后续步骤使用。
   - **Safe HTTP fetch（安全拉取）**：同样是 HTTP 请求拉取数据；**即使请求失败，也不会让整个工作流失败**，失败可被后续逻辑处理或忽略（具体语义以产品/引擎为准）。
4. **其他节点类型**（如逻辑判断、转换、延时、以及调色板里出现的各类插件）：各配 **一行用途说明**，与真实执行行为一致。
5. **展示**：说明在调色板内可读（标题下副标题、第二行小字等），保持紧凑、可扫读；遵循当前设计体系（如 Carbon）。

### 范围

- **前端 `workflow-ui`**：画布左侧节点列表 / 调色板、节点元数据（描述字段）；若描述需来自配置/API，在 Arch 中说明。  
- **不涉及**：本 T2 文档条目不包含任何代码提交；实现时另开正式 TODO / 版本文档。

### 验收方向（供后续 test-doc 展开）

- 调色板中每个节点类型均可见说明；文案与引擎行为无矛盾。  
- 拖拽添加节点仍可用；移动端「添加节点」列表展示相同或等价说明。
